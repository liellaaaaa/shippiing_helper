## 1. 概述

### 1.1 模块目标

支持船务人员快速计算外贸出货的包装方案（桶数、卡板数、总体积、总毛重），并智能推荐最经济的集装箱柜型（20GP / 40GP / 40HQ）。同时兼容空运计费重量和陆运总重阈值计算。

### 1.2 核心定位

**FR-4.x = 物流测算参谋（只读计算，不影响订单数据）**

- 职责：根据输入的订单量和包装规格，快速输出装配方案和集装箱推荐
- 职责边界：不写入任何数据，仅作为测算工具
- 后续流程：用户根据计算结果自行安排订舱和出运

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| What-if 即时反馈 | 用户调整参数时实时重新计算，形成流畅的沙盘推演体验 |
| 业务翻译优先 | 将数学公式翻译为可执行的业务指令（"建议使用 40GP，满载率 92%"） |
| 三态兼容 | 海运（复杂）、空运（精简）、陆运（基础）一套界面统一管理 |
| 零数据侵入 | 本模块仅读取和计算，不写入 orders 或 pi_contracts 表 |

## 3. 计算公式

### 3.1 海运计算

**桶/箱数量**：
```
drums = ⌈order_quantity_kg / net_kg_per_drum⌉
```

**卡板数量**（向上取整）：
```
pallets = ⌈drums / capacity_per_pallet⌉
```

> 不打卡板时：pallets = 0

**总体积**：
```
total_cbm = (drums × cbm_per_drum) + (pallets × cbm_per_pallet)
```

**总毛重**：
```
total_weight_kg = (drums × gross_per_drum) + (pallets × pallet_weight)
```

> 不打卡板时：pallets 相关项为 0

**20GP 判定**：
```
if total_cbm ≤ 28 AND total_weight_kg ≤ 21,000 → "适合 20GP"
elif total_cbm ≤ 56 AND total_weight_kg ≤ 27,000 → "建议升级 40GP"
elif total_cbm ≤ 67 AND total_weight_kg ≤ 27,000 → "建议升级 40HQ"
else → "⚠️ 超出单柜极限，建议拆单或换船"
```

**装载率**：
```
load_rate = total_cbm / container_cbm × 100%
```

### 3.2 空运计算

**体积重（IATA ×167）**：
```
vol_weight_167 = total_cbm × 167
```

**体积重（航司 ÷6000）**：
```
vol_weight_6000 = total_cbm × 1000000 / 6000
```

**计费重量**：
```
chargeable_weight = max(actual_weight, vol_weight_167, vol_weight_6000)
```

### 3.3 陆运计算

**总件数**：
```
total_drums = ⌈order_quantity_kg / net_kg_per_drum⌉
```

**公路限重阈值**：总重超过 30,000 kg 时给出警告提示。

## 4. 功能需求

### FR-4.1：输入模式切换

**顶部控制面板**：
- Toggle 开关：【📦 按订单计算】 / 【✍️ 手动输入】
- 默认显示"按订单计算"模式

**按订单计算模式**：
- 下拉搜索框：输入订单号/客户名搜索，选中订单后展示产品列表
- 选择产品后，自动填充 quantity_kg
- 根据 internal_code 匹配 products_knowledge 表，自动推荐包装类型，显示推荐理由气泡

**手动输入模式**：
- 数字输入框：订单量（kg）
- 下拉选择：包装类型（13 种）
- 下拉选择：卡板规格（1.0×1.0m / 1.1×1.1m）
- 数字输入框：单板数量（自动填充默认值，允许人工修改）
- 复选框：不打卡板

**URL 联动**：
- 从 FR-3.x 跳转时携带 `?mode=order&internal_code=XXX`
- 前端检测到参数后，自动激活"按订单计算"模式并带入数据

### FR-4.2：运输模式切换

**顶部 Radio Group**：【🚢 海运 | ✈️ 空运 | 🚛 陆运】（默认海运）

**海运模式**：展示完整计算区（桶数、卡板数、体积、毛重、集装箱推荐）

**空运模式**：
- 隐藏：卡板数、集装箱推荐
- 新增：计费重量对比面板（物理实重 / IATA×167 / 航司÷6000）+ 取大者标红

**陆运模式**：
- 隐藏：卡板数、集装箱推荐
- 新增：总件数 + 公路限重阈值警告

### FR-4.3：包装类型智能推荐

**触发时机**：用户在"按订单计算"模式下选中一个产品后

**逻辑**：
1. 根据 internal_code 查询 products_knowledge 表
2. 找到该产品的历史包装类型，自动填充到包装类型下拉框
3. 下拉框高亮闪烁一次提示用户
4. 显示推荐理由气泡："💡 系统根据产品知识库自动推荐：125kg新款胶桶"

**人工接管**：
- 用户手动修改包装类型后，推荐理由气泡立即消失
- 后续计算以用户选择的为准

### FR-4.4：实时计算

**触发时机**：任何输入变化后，自动重新计算

**防抖机制**：前端延迟 300ms，输入停止后再发 API 请求

**加载态**：计算中时，结果区域显示骨架屏或透明度降低

**空状态保护**：必填项未填完时，显示友好空状态提示，不抛红色报错

### FR-4.5：不打卡板选项

**UI 交互**：
- 勾选复选框后：卡板规格选择器 + 单板数量输入框 → Disabled
- 卡板数量结果卡片：数字 → 0，附加小标签"未使用"
- 文本装配方案自动改写："已选择不打卡板。共需 X 个桶，按散货堆叠..."

**数据连续性**：数据结构保持不变，体积和重量仅基于桶身数据累加

### FR-4.6：计算结果展示

**区域一：智能结论区（顶部高亮背景）**
```
✅ 完美整除，无尾数。共需 150 个桶，满板 10 个，每板 15 个。
当前体积约 8.42 CBM，强烈建议使用 20GP（装载率 30%）。
```

**区域二：核心指标卡片组（中部网格）**

| 产品总量 | 所需桶数 | 所需卡板数 | 总毛重(KG) | 总体积(CBM) |
|---------|---------|---------|-----------|-----------|
| 1500 kg | 150 | 10 | 15,750 | 8.42 |

**区域三：集装箱推荐卡片**
- 柜型图标（20GP / 40GP / 40HQ / 超限警告）
- 装载率："预计占用 25 CBM / 28 CBM，装载率 89%"

**区域四：详细参数溯源（底部折叠面板）**
- 单桶尺寸、CBM、皮重
- 单卡板容量、CBM、重量
- 计算中间值

### FR-4.7：空运计费重量面板

**并排展示**：
- 📏 物理实重：XXX KG（灰色基准）
- ✈️ IATA 标准 (×167)：YYY KG（绿色，"国际通用标准"）
- 📦 航司标准 (÷6000)：ZZZ KG（蓝色，"部分航司标准"）

**智能提示**：
```
⚠️ 实际计费重量取决于您合作的货代或航空公司报价条款，请以最终确认的系数为准。
```

**取大者标红**：自动找出三个值中的最大值并高亮显示。

## 5. API 接口

### 5.1 包装计算

#### GET /api/v1/packages/calculate

**描述**：计算包装方案和集装箱推荐

**查询参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| mode | string | 是 | `order` / `manual` |
| quantity_kg | float | 是 | 订单量（kg） |
| packaging_type | string | 是 | 包装类型名称 |
| pallet_spec | string | 否 | `1.0x1.0` / `1.1x1.1`（默认 `1.1x1.1`） |
| pallet_qty | int | 否 | 单板数量（自动取默认值） |
| no_pallet | bool | 否 | 不打卡板（默认 false） |
| transport_mode | string | 否 | `sea` / `air` / `land`（默认 `sea`） |
| order_id | int | 否 | 订单ID（mode=order 时传入） |
| internal_code | string | 否 | 内部编码（用于知识库匹配） |

**响应（sea 模式）**：
```json
{
  "drums": 150,
  "pallets": 10,
  "total_cbm": 8.42,
  "total_weight_kg": 15750.0,
  "packing_scheme": "共需 10 个卡板，前 9 个满板（每板 15 个），最后一个板装 15 个。",
  "container": {
    "recommended": "20GP",
    "load_rate": 30.0,
    "volume_limit": 28.0,
    "weight_limit": 21000,
    "status": "ok"
  },
  "packaging": {
    "name": "125kg新款胶桶",
    "drum_cbm": 0.21,
    "drum_tare_kg": 6.0,
    "drum_gross_kg": 131.0,
    "pallet_spec": "1.1x1.1",
    "pallet_capacity": 5
  }
}
```

**响应（air 模式）**：
```json
{
  "actual_weight_kg": 15750.0,
  "vol_weight_167": 1408.2,
  "vol_weight_6000": 1408.3,
  "chargeable_weight_kg": 15750.0,
  "chargeable_weight_note": "取大者"
}
```

**响应（land 模式）**：
```json
{
  "total_drums": 150,
  "total_weight_kg": 15750.0,
  "total_cbm": 8.42,
  "overweight_warning": false
}
```

### 5.2 包装类型列表

#### GET /api/v1/packages/types

**描述**：获取所有包装类型（用于下拉选择）

**响应**：
```json
{
  "types": [
    {
      "id": 9,
      "name": "125kg新款胶桶",
      "net_kg": 125.0,
      "cbm": 0.21,
      "pallet_1x1": 4,
      "pallet_1_1x1_1": 5
    }
  ]
}
```

### 5.3 知识库推荐

#### GET /api/v1/packages/recommend?internal_code={code}

**描述**：根据 internal_code 查询知识库推荐包装类型

**响应**：
```json
{
  "internal_code": "SILI-001",
  "recommended_packaging": "125kg新款胶桶",
  "reason": "历史使用记录"
}
```

## 8. 非目标

- 本模块不写入任何数据到数据库
- 不支持多产品批量计算（每次只计算一个包装类型）
- 不支持海运拼柜（单柜最大装载量计算）
- 不支持空运和陆运的复杂嵌套计算（仅基础适配）
